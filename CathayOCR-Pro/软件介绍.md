# 古籍 OCR 助手 — PDF 批量转文字工具

> **不用学编程，不用配环境。双击打开，拖入 PDF，点一下开始，等几分钟拿到文字。**

---

## 这是干什么的？

你有一批古籍/文献的 **PDF 扫描件**（或者图片扫描成的 PDF），想把里面的**文字提取出来**变成可以搜索、复制、编辑的文本文件。

传统做法是：打开 PDF → 截图 → 一张张手动打字 → 太累了。

这个工具帮你**全自动做完**——你只需要告诉它哪几个 PDF 文件要处理，剩下的事它自己干。

---

- 📖 **文史爱好者** — 整理家谱、旧书、手稿
- 🌍 **多语言文献工作者** — 处理法文、德文、日文、俄文、韩文、阿拉伯文等多语种 PDF

## 怎么安装？

**什么都不用装。** 便携包里自带 Python 3.10、CUDA 库、所有依赖库和 OCR 模型，解压完直接运行。

```
1. 下载 OCR-Proxy-Portable.zip（约 9 GB）
2. 右键 → 解压到当前文件夹（建议解压到 SSD 硬盘）
3. 打开解压后的文件夹，找到「启动.bat」
4. 双击它
5. 控制台窗口会先检测 CUDA（有 NVIDIA 显卡会提示可用，没有也不影响）
6. 等 10~30 秒，主界面弹出来就可以用了
```

> ℹ **关于 CUDA**：便携包已内置 CUDA/cuBLAS/cuDNN DLL（约 2.4 GB）。有 NVIDIA 显卡的机器插上就能用 ONNX CUDA 引擎，无需单独安装 NVIDIA CUDA 工具包。
>
> ⚠ **如果杀毒软件报毒**：这是正常的误报，因为程序里调用了命令行程序和 Python 脚本。点击「允许运行」即可。

## 怎么用？超详细手把手教程

### 第一步：打开程序

双击 `启动.bat`，会弹出一个黑色控制台窗口（放着别关，最小化就行），然后主界面会出现。

### 第二步：选择模式（🎯 简单模式  vs  🔧 专业模式）

界面左上角有两个选项：

- **🎯 简单模式**（**推荐新手**）— 回答 3 个问题 + 选个语言，系统自动配好一切参数
- **🔧 专业模式** — 每个参数自己调，适合有经验的高级用户

**刚开始用？直接默认就是🎯简单模式，继续往下看。**

### 第三步：简单模式三问（10 秒搞定）

#### ① 文档类型

| 选哪个 | 什么时候选 |
|-------|-----------|
| 📜 古籍/校勘 | 竖排繁体、校勘记、地方志——**古籍类文档必选** |
| 📄 普通文档 | 横排印刷品、现代书籍、论文 |
| 🗂️ 扫描件 | 没有 OCR 层的扫描件、拍照文档 |

#### ② 精度/速度

| 选哪个 | 效果 | 速度 |
|-------|------|------|
| 🎯 精度优先 | 最准确 | 慢一些 |
| ⚖️ 平衡（默认） | 够用 | 快 |
| 🚀 速度优先 | 还行 | **最快** |

#### ③ 显卡选择

| 选哪个 | 什么意思 |
|-------|---------|
| NVIDIA ≥12 GB | 大显存 NVIDIA（RTX 3060 以上）——可以用最高精度模式 |
| NVIDIA ≤8 GB | 小显存 NVIDIA（RTX 3050/5060 等）——精度优先时自动降低分辨率防爆显存 |
| AMD / Intel 独显 | AMD Radeon 或 Intel Arc 独立显卡 |
| 纯核显 / CPU | 没独显，或者用 AMD Vega / Intel UHD 之类 |
| 🤷 我不知道 | 系统自动检测，选最优配置 |

#### ④ 文档语言（🌐 选语言）

总共 **75 种语言/文字选项**，按你 PDF 里是什么文字选就行：

**常用的几个：**
- **中文 (Chinese)** — 繁简体通用，一个选项搞定
- **English (英文)、Français (法文)、Deutsch (德文)、日本語 (日文)**
- **Русский (俄文)、한국어 (韩文)**
- **العربية (阿拉伯文)、हिन्दी (印地文)、ภาษาไทย (泰文)**
- 其他 70+ 种欧洲/亚洲语言全部在下拉框里

> 💡 **注意**：选韩文/俄文时系统会自动切到 EasyOCR 引擎；选阿拉伯/印地/泰文时自动切到 PP-OCRv6 ONNX CUDA 引擎。其他语言默认用最快的 ncnn Vulkan 引擎。

### 第四步：添加文件

把 PDF 文件**拖进去**（直接拖到文件列表空白区域），或者点击「添加PDF文件」按钮选文件。可以一次选多个。

### 第五步：选输出目录

点击「选择输出目录」，告诉程序把识别结果放在哪里。不选的话默认放在 PDF 文件同目录。

### 第六步：点「开始处理」

然后等着就行。进度条会告诉你处理了多少。

### 第七步：看结果

处理完成后，输出目录里会出现：
- `XXX_result.txt` — 识别的纯文字（记事本打开就能看）
- `XXX_layered.pdf` — 带文字层的 PDF（可以在里面选中文字、复制、搜索）

## 快速参考

### 引擎怎么选？

| 你的电脑情况 | 引擎选这个 | 备注 |
|------------|-----------|------|
| **有 NVIDIA 显卡**（GTX/RTX 系列） | ⭐ **ncnn Vulkan** | ⭐首推！速度最快，什么显卡都能用 |
| **AMD / Intel 独显** | ⭐ **ncnn Vulkan** | 唯一 GPU 加速选项 |
| **有 NVIDIA，但 PDF 含阿拉伯/印地/泰文** | **PP-OCRv6 ONNX CUDA** | 便携版自带 CUDA，即插即用 |
| **有 NVIDIA，但 PDF 含韩文/俄文** | **EasyOCR** | 韩文俄文识别效果最好 |
| **没独显 / 不知道有没有显卡** | **ncnn Vulkan** + 模式选"自动" | 有 GPU 自动用，没 GPU 自动切 CPU |

> 💡 **不知道有没有独立显卡？** 打开任务管理器（Ctrl+Shift+Esc）→ 性能 → 看左边有没有多个「GPU」。如果有 GPU 0 和 GPU 1，说明有独立显卡。

### 语言选哪个？

| 你的 PDF 是什么文字 | 选这个 |
|-------------------|--------|
| 中文古籍（繁简体都行） | **中文 (Chinese)** — 一个选项覆盖繁简体 |
| 日文古籍 / 日本文献 | **日本語 (日文)** |
| 韩文文献 | **한국어 (韩文)** — 自动用 EasyOCR |
| 法文 / 德文 / 西班牙文 / 意大利文 | 字母语言各自选对应项 |
| 俄文 / 乌克兰文 | **Русский (俄文)** / українська — 自动用 EasyOCR |
| 阿拉伯文 / 波斯文 / 维吾尔文 | 阿拉伯文系选一个 |
| 印地文 / 尼泊尔文 / 梵文 | 天城文系选一个 |
| 混合多种语言（中英法德日韩俄） | **多语言 (v6)** |
| 欧洲 46 种语言混合 | **多语言 (v5)** |

## 文件结构说明

解压后你会看到这些文件夹：

```
OCR-Proxy-Portable/
├── 启动.bat           ← 🚀 入口！双击这里开始
├── CathayOCR-Pro/     ← 主程序（不动它）
├── portapython/       ← 内置 Python（不动它，删了程序跑不起来）
├── ncnn/              ← ncnn 引擎 + 模型（主力引擎）
├── ppocr_v6/          ← ONNX CUDA 引擎 + 模型（备选主力）
├── easyocr_data/      ← EasyOCR 模型（韩文/俄文用）
├── easyocr_universal/ ← EasyOCR 启动脚本
├── easyocr_french/    ← EasyOCR 服务器脚本
├── ppocr_v3/          ← 备胎引擎（可以删，260 MB）
├── ppocr_v5/          ← 备胎引擎（可以删，400 MB）
└── 软件介绍.md        ← 本文档
```

> 如果硬盘空间紧张：可以删除 `ppocr_v3/` 和 `ppocr_v5/` 两个备胎引擎文件夹，节省约 700 MB。运行不受影响。

## 常见问题（小白版）

### Q: 打开后黑框一闪就没了？
A: 解压不完整。确认 `portapython\python.exe` 文件存在，不要单独移动任何文件夹。

### Q: 进度不动了？
A: 点「停止」按钮，关掉程序，重新打开再试。偶尔单个页面比较难识别会卡住。

### Q: 识别出来的文字好多错误？
A: 检查下面几个：
1. 渲染倍率调到 **3x**
2. 图像边长调到 **3000**
3. 模型选 **v6 Server**

### Q: 速度太慢了？
A: 试试这几个：
1. 确认模式选了"自动"或"GPU模式"
2. 开启「双实例并行」
3. 精度选 FP16

### Q: 竖排文字识别不好？
A: 确认「竖排文字」是打勾的。如果还不好，图像边长加到 3000，渲染倍率调到 3x。

### Q: 这个和普通的 Umi-OCR 有什么区别？
A: 普通 Umi-OCR 更适合单张图片或少量文字的识别。这个工具是专门为**大量 PDF 文件**设计的，多了自动流水线、双实例并行加速、GPU 设备选择、简单模式等功能，批处理效率高得多。

## 简单说人话版

把这个工具想象成一个**自动抄书工**：

- 你把 PDF 书页丢给它
- 它自动翻页、拍照、认字、抄写
- 抄完后给你一份纯文字的电子版
- 你可以复制、搜索、修改里面的文字

它的速度取决于你给它配的"眼镜"（引擎和模型）：
- ⭐ **ncnn Vulkan** = ⭐首推！什么显卡都能加速，没显卡自动切 CPU，到哪里都能用
- **PP-OCRv6 (ONNX CUDA)** = 自带 CUDA 库，有 NVIDIA 显卡的电脑插上就能用
- **EasyOCR** = 专治韩文、俄文
- **ncnn CPU** = 什么电脑都能用

> **还不确定？选 🎯 简单模式，一切默认，开干就行。**
