         │               │                     │

    │  │             │  │  ├── NcnnCPU         │      │
    │  │             │  │  └── EasyOCR Pipe    │      │

**设计理念**：GPU 永不等待，CPU 预渲染消除 I/O 瓶颈。CPU 在后台持续预渲染 PDF 页面并压入队列，OCR 引擎从队列中消费，二者全速并行。

## 核心特性

### 引擎管理（6 引擎统一注册）
通过 `ENGINE_REGISTRY` 统一管理，每个引擎声明入口文件、协议类型（管道 / TCP 持久连接 / 子进程）、GPU/CPU 支持、语言范围、可用参数。

| 适配器模式 | 说明 | 对应引擎 |
|-----------|------|---------|
| `PaddlePipeAdapter` | 管道 JSON 协议，子进程标准输入输出 | PP-OCRv6 ONNX CUDA、EasyOCR、PP-OCRv5、PP-OCRv3 |
| `NcnnVulkanAdapter` | TCP 持久连接，Vulkan GPU 加速 | ncnn Vulkan |
| `NcnnCPUAdapter` | 子进程单次调用，纯 CPU | ncnn CPU |

### 双模式界面
- **🎯 简单模式**：只需回答 4 个问题（文档类型、精度速度、显卡、语言），自动映射全部专业参数。小白用户 10 秒完成配置。
- **🔧 专业模式**：全部参数自由调节，适合有经验的高级用户。

### 双实例并行
启动两个独立 OCR 子进程轮询调度，GPU 模式下效果显著（Vulkan 可翻倍），CPU 模式下提升有限。CPU 模式自动禁用。

### GPU 加速
- **ncnn Vulkan**：跨厂商 GPU 支持（NVIDIA / AMD / Intel 任意品牌）。原生 CPU 回退，无 GPU 也能跑。
- **ONNX Runtime CUDA**：NVIDIA 专属加速。便携版已内置 CUDA DLL，无需系统安装 CUDA 工具包，即插即用。
- 启动时自动检测 GPU 设备，优先选中独立显卡。
- FP16 半精度推理（ncnn 引擎），显存占用减半，速度提升约 30%。

### 多语言支持
75 种语言/文字选项，涵盖 CJK、拉丁语系、西里尔语系、希腊文、阿拉伯文系、天城文系、东南亚文字（详见[语言支持范围](#语言支持范围)）。

### 容错机制
- **超时保护**：单页 OCR 超过 180 秒自动跳过
- **安全定时器**：整体任务无进度超过 300 秒触发紧急停止
- **进程清理三保险**：SIGTERM → `taskkill /f` → `atexit` 兜底
- **快速取消**：直接终止子进程，秒级响应

### 各引擎一句话总结

| 引擎 | 什么时候用 |
|------|-----------|
| ⭐ **ncnn Vulkan** | **默认首选**。速度最快，任意品牌显卡都能 GPU 加速，没显卡自动切 CPU。v3~v6 全系列模型随便换。 |
| **PP-OCRv6 ONNX CUDA** | 需要识别阿拉伯文、天城文（印地/尼泊尔/梵文）、泰文等 V5 语系文字时使用。便携版自带 CUDA 库，即插即用。 |
| **EasyOCR** | 需要识别韩文 / 俄文时使用（便携版已缓存模型，开箱即用）。 |
| **PP-OCRv5/v3 (Paddle CPU)** | 备胎引擎，其他引擎都用不了时顶上。 |

## 引擎对比与选型指南

| 评价维度 | ncnn Vulkan ⭐ | PP-OCRv6 ONNX CUDA | ncnn CPU |
|---------|:------------:|:-----------------:|:--------:|
| **速度** | ⭐⭐⭐⭐⭐ 最快 | ⭐⭐⭐⭐ 快 | ⭐⭐ 慢 |
| **识别精度** | ⭐⭐⭐⭐⭐（同模型） | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ |
| **显卡兼容** | 任意 GPU | 仅 NVIDIA | 不要显卡 |
| **CPU 回退** | ✅ 原生 | ✅ 支持 | ✅ 原生 |
| **双实例效果** | 优秀（显存低） | 好（显存高） | 一般 |
| **双实例速度** | ~2.2 页/秒 | ~1.1 页/秒 | ~0.4 页/秒 |
| **模型兼容** | v3~v6 全系列 | 固定 v6 | v3~v6 全系列 |
| **语言覆盖** | V6 语系（60+ 种） | **全部**（64+ 种） | V6 语系 |

> ⚠ **重要说明**：ncnn Vulkan 和 PP-OCRv6 ONNX CUDA 都用 PP-OCRv6 模型，**识别精度由所选模型版本决定，不取决于引擎**。同模型下 ncnn Vulkan 识别率略高。

### 简单场景快速选择

| 你的情况 | 推荐引擎 | 理由 |
|---------|---------|------|
| **有 NVIDIA 显卡，只是普通文档（中/英/法/德/日）** | ⭐ **ncnn Vulkan** | 速度最快，兼容最好 |
| **有 NVIDIA 显卡，含阿拉伯/印地/泰/希腊文** | **PP-OCRv6 ONNX CUDA** | V5 语系只有它支持 |
| **AMD / Intel 独显** | **ncnn Vulkan** | 唯一 GPU 选项 |
| **需要识别韩文 / 俄文** | **EasyOCR** | 识别效果好于 PP-OCR |
| **无独显 / 纯核显** | **ncnn CPU** 或 PP-OCRv5 | 稳定兼容 |
| **不确定配置，就想快点上手** | ⭐ **ncnn Vulkan** + 简单模式 | 全自动 |

### 操作步骤

#### 简单模式（推荐新手）

界面左上角勾选「🎯 简单模式」，依次回答：

1. **文档类型** 📄：古籍/校勘、普通文档、扫描件
2. **精度/速度** ⚡：精度优先、平衡（默认）、速度优先
3. **显卡选择** 🖥️：NVIDIA≥12GB / NVIDIA≤8GB / AMD或Intel独显 / 纯核显或CPU / 我不知道
4. **文档语言** 🌐：选对应文字的语言（75 种可选）

系统自动映射引擎、GPU、边长、渲染倍率、精度、双实例等全部参数。点「开始处理」即可。

#### 专业模式（高级用户）

勾选「🔧 专业模式」后，可自由调节全部参数：

1. **选引擎**：推荐的 ncnn Vulkan 已默认选中，可切换
2. **选模式**：自动(GPU优先) / GPU模式 / CPU模式
3. **选语言和模型**：语言根据引擎自动过滤可用选项
4. **调整参数**：边长、渲染倍率、精度、竖排文字、批处理数等
5. **添加文件**：拖拽或点击「添加PDF文件」
6. **选择输出目录**（可选，默认在 PDF 同目录）
7. **点击「开始处理」**：等待进度 100%

### 输出文件

每个 PDF 处理完成后，输出目录生成：

- `原文件名_result.txt` — 纯文本识别结果（每个 PDF 一个文件）
- `原文件名_layered.pdf` — 含文本层的 PDF（可搜索、复制）

## 参数参考

| 参数 | 说明 | 范围 | 推荐值 |
|------|------|------|--------|
| **图像边长** | OCR 输入的长边像素值。越大细节越多，速度越慢。6 号小字≥2000。 | 320~6400 | **2000** |
| **渲染倍率** | PDF 渲染相对于原始分辨率的缩放。 | 1x / 2x / 3x | **2x** |
| **精度** | 浮点精度（仅 ncnn 引擎生效）。FP16 快 30%，显存减半。 | FP32 / FP16 | **FP32**（精度）<br>**FP16**（速度） |

| **批处理数** | 每次并发识别的图片数。越大 GPU 利用率越高，显存占用线性增长。 | 1~64 | 12（GPU）<br>6（CPU） |
| **双实例并行** | 启动两个 OCR 子进程轮询。GPU 模式效果显著。 | 开/关 | **GPU：开** |
| **模型** | OCR 模型版本（ncnn 引擎）。 | v3~v6 | **v6 Medium** |
| **GPU 设备** | 手动指定 Vulkan 设备索引。 | 自动 / 各 GPU | **自动** |

## 常见问题

### Q: 双击启动.bat 后黑框一闪而过？
A: 解压不完整。确认 `portapython\python.exe` 存在，不要单独移动文件或修改目录结构。

### Q: 启动时提示"未检测到 CUDA"？
A: 这是正常提示（没有 NVIDIA 显卡或没有驱动时才出现）。选 ncnn Vulkan 引擎一样能 GPU 加速（AMD/Intel 显卡都可以）。

### Q: 进度不动了 / 卡住了？
A: 点击「停止」，关掉程序重新打开。如果频繁卡住：① 检查任务管理器里有没有残留的 OCR 进程；② 调低批处理数；③ 选 ncnn CPU 引擎试试。

### Q: 识别结果全是乱码？
A: 检查语言是否选对。如果选了中文但 PDF 是法文，请改语言。v6 模型引擎必须使用配套的 `ppocr_keys_v6.txt` 字典。

### Q: 速度太慢？
A: ① 确认模式是"自动"或"GPU模式"；② 打开「双实例并行」；③ 精度选 FP16；④ 调低渲染倍率到 1x。

### Q: 识别质量不好？
A: ① 渲染倍率调到 3x；② 图像边长调到 3000；③ 选 ncnn CPU 引擎（稳定）；④ 如果用 ONNX CUDA，试一下模型从 small 换 medium。

### Q: GPU 利用率很低（<50%）？
A: 开启双实例并行，提高批处理数（12~16）。Vulkan 双实例可以吃满 GPU。

### Q: 便携包太大（9 GB），能删些什么？
A: 可以安全删除的是：
- `ppocr_v3/` (300 MB) 和 `ppocr_v5/` (405 MB)——备胎引擎，大多数情况用不到
- `ncnn/models/` 里不需要的模型版本（v3/v4 约 100 MB 每个）
- `easyocr_data/` (1 GB) 删掉后韩文/俄文识别会降级，非必需
- **不能删**：`portapython/`（运行核心）、`ppocr_v6/`（主力引擎）、`ncnn/`（主力引擎）

### GPU 兼容性速查

| GPU | ncnn Vulkan | PP-OCRv6 ONNX CUDA | 推荐 |
|-----|:-----------:|:------------------:|------|
| NVIDIA RTX 系列 | ✅ | ✅（内置 DLL 即插即用） | **ncnn Vulkan**（速度优先）或 ONNX CUDA（V5 语系） |
| NVIDIA GTX 系列 | ✅ | ✅ | ncnn Vulkan |
| AMD Radeon RX 5000+ | ✅ | ❌ | ncnn Vulkan |
| Intel Arc A 系列 | ✅ | ❌ | ncnn Vulkan |
| AMD Vega / 核显 | ❌ | ❌ | ncnn CPU |
| Intel UHD / Iris Xe | ❌ | ❌ | ncnn CPU / PP-OCRv5 |
